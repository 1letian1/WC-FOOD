请为我设计一套完整的微信点餐小程序产品原型，产品支持“堂食”和“配送”两种订单模式，包含用户端和商家端。

请按照真实可开发的微信小程序标准进行设计，不要只生成概念展示图。需要完整考虑页面结构、业务流程、订单状态、组件复用、表单交互、异常状态、空状态、按钮状态以及页面之间的跳转关系。

一、项目基本信息

产品名称暂定为“食刻到家”。

产品类型：微信点餐小程序。

使用场景：该小程序主要供少量固定用户和一个商家使用。

主要角色：

1. 普通用户
2. 商家

当前配送模式：

所有配送订单均由商家自行配送，不接入第三方配送平台，不需要骑手端，不需要地图实时轨迹。

支持的订单模式：

1. 堂食
2. 配送

堂食用户到店用餐或取餐。

配送用户填写地址，由商家制作完成后自行配送。

二、设计目标

请设计一套结构清晰、操作简单、适合实际开发的微信小程序原型。

原型需要满足以下目标：

1. 用户能够快速选择堂食或配送。
2. 用户能够浏览商品、加入购物车并提交订单。
3. 商家能够及时看到新订单并完成接单。
4. 商家能够根据订单类型处理堂食和配送订单。
5. 用户能够查看订单状态。
6. 商家能够管理商品和店铺营业状态。
7. 所有页面保持统一的设计规范。
8. 页面结构需要符合微信小程序的交互习惯。
9. 所有关键业务必须形成完整闭环。
10. 页面应适合后续使用微信小程序原生开发、uni-app或Taro进行实现。

三、设计尺寸与布局规范

请按照微信小程序移动端界面设计。

基础画板尺寸：

宽度：375px
高度：812px

顶部需要预留微信状态栏和小程序导航栏区域。

建议页面结构：

1. 微信状态栏
2. 小程序顶部导航栏
3. 页面主要内容区域
4. 底部操作栏或TabBar
5. 适配安全区域

页面设计要求：

1. 内容区域左右边距为16px。
2. 卡片圆角建议为12px。
3. 主按钮高度建议为44px至48px。
4. 普通输入框高度建议为44px。
5. 页面元素之间保持8px、12px、16px或24px的统一间距。
6. 底部操作按钮需要避开手机安全区域。
7. 底部TabBar固定显示。
8. 长页面允许纵向滚动。
9. 页面不能出现横向滚动。
10. 弹窗、抽屉和底部弹出层需要符合微信小程序交互习惯。

七、用户端完整业务流程

用户登录后进入首页。

用户首先选择订单模式：

1. 堂食
2. 配送

用户选择模式后浏览商品。

用户可以：

1. 切换商品分类
2. 搜索商品
3. 查看商品详情
4. 选择商品规格
5. 增加或减少数量
6. 加入购物车
7. 查看购物车
8. 修改商品数量
9. 删除商品
10. 清空购物车
11. 去结算

堂食订单流程：

用户选择堂食
→ 浏览商品
→ 加入购物车
→ 去结算
→ 填写桌号或选择“暂未入座”
→ 填写联系电话
→ 填写订单备注
→ 提交订单
→ 商家接单
→ 商家开始制作
→ 商家标记待取餐或待用餐
→ 用户查看订单状态
→ 商家完成订单

配送订单流程：

用户选择配送
→ 浏览商品
→ 加入购物车
→ 去结算
→ 选择收货地址
→ 填写联系电话
→ 查看商家配送说明
→ 填写订单备注
→ 提交订单
→ 商家接单
→ 商家开始制作
→ 商家开始配送
→ 用户查看配送状态
→ 商家标记送达
→ 用户确认收货
→ 订单完成

十九、特别注意

1. 堂食订单不能显示配送地址和配送费。
2. 配送订单必须显示收货地址和联系电话。
3. 堂食订单不能出现“配送中”状态。
4. 配送订单不能出现“待取餐”状态。
5. 用户首页切换堂食和配送后，结算页面必须同步变化。
6. 商家订单列表必须明显区分堂食订单和配送订单。
7. 商家工作台需要分别显示堂食待取餐数量和配送中数量。
8. 用户端和商家端的订单状态必须完全对应。
9. 不要设计骑手端。
10. 不要设计第三方配送平台功能。
11. 不需要优惠券、积分、会员等级、拼团、秒杀等营销功能。
12. 不需要复杂的库存系统。
13. 第一版优先保证点餐、接单、制作、堂食、配送和订单完成流程可以正常闭环。
14. 所有页面都要考虑加载中、空数据、失败和禁用状态。
15. 页面应具备真实产品感，不能只像静态展示稿。
