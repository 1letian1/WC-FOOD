# 微信点餐小程序产品需求整理文档

## 1. 项目定位

### 1.1 项目名称

暂定名称：食刻小馆

### 1.2 项目类型

微信点餐小程序。

### 1.3 项目定位

本项目是一个面向少量固定用户和一个商家的轻量级点餐小程序，主要用于解决用户在线点餐、商家接单、堂食处理和商家自配送的问题。

项目不追求复杂平台化能力，第一版重点是把点餐、接单、制作、堂食、配送和订单完成这条主流程跑通。

### 1.4 使用场景

用户可以通过微信小程序选择堂食或配送，浏览商品并下单。

商家可以通过同一个小程序的商家登录入口进入商家端，处理订单、管理商品、设置店铺营业状态。

### 1.5 核心特点

- 支持堂食和配送两种下单方式
- 配送由商家自配送，不接入第三方配送平台
- 商家和用户共用同一个微信小程序
- 普通用户使用微信快捷登录
- 商家使用账号密码登录
- 不设计骑手端
- 不设计邀请码绑定
- 不设计微信自动识别商家身份
- 第一版以简单、稳定、可开发为主

## 2. 项目目标

### 2.1 用户目标

用户可以方便地完成以下事情：

- 浏览商品
- 选择堂食或配送
- 加入购物车
- 提交订单
- 查看订单状态
- 取消未接单订单
- 查看历史订单
- 配送订单确认收货

### 2.2 商家目标

商家可以方便地完成以下事情：

- 登录商家端
- 查看今日订单情况
- 接收新订单
- 接单或拒单
- 处理堂食订单
- 处理配送订单
- 管理商品
- 设置商品上下架
- 设置店铺营业状态
- 退出商家登录

### 2.3 系统目标

系统需要保证：

- 用户端和商家端流程清晰分离
- 堂食订单和配送订单状态不混乱
- 商家端操作按钮根据订单状态动态变化
- 商品上下架、售罄、店铺休息等异常情况有提示
- 页面结构适合后续用微信小程序、uni-app或Taro开发

## 5. 登录方案

## 5.1 采用方案

本项目采用方案三：

普通用户微信快捷登录，商家账号密码登录。

### 登录结构

```text
统一登录页
├── 普通用户：微信快捷登录 → 用户首页
└── 商家人员登录 → 商家登录页 → 商家工作台
```

---

## 5.2 普通用户登录流程

```text
用户打开小程序
→ 进入统一登录页
→ 勾选用户协议和隐私政策
→ 点击微信快捷登录
→ 登录成功
→ 进入用户首页
```

### 登录异常

- 未勾选协议时，微信登录按钮不可用
- 用户取消授权时，提示“已取消微信授权”
- 登录失败时，提示“登录失败，请稍后重试”
- 网络异常时，提示“网络连接失败，请稍后重试”

---

## 5.3 商家登录流程

```text
用户打开小程序
→ 进入统一登录页
→ 点击商家人员登录
→ 进入商家登录页
→ 输入商家账号和密码
→ 点击登录商家端
→ 登录成功
→ 进入商家工作台
```

### 商家登录页字段

- 商家账号
- 登录密码
- 记住账号
- 忘记密码
- 登录商家端按钮
- 返回用户登录

### 商家登录异常

- 账号为空：请输入商家账号
- 密码为空：请输入登录密码
- 账号或密码错误：账号或密码错误，请重新输入
- 账号被禁用：当前账号已被禁用，请联系管理员
- 网络异常：网络连接失败，请稍后重试
- 登录失败次数过多：登录失败次数过多，请稍后再试

### 忘记密码处理

点击忘记密码后弹窗提示：

请联系店铺管理员重置密码。

- 已送达
- 已完成

- 已拒单

---

## 7.2 堂食订单状态流转

```text
待接单
→ 已接单
→ 制作中
→ 待取餐
→ 已完成
```

### 堂食异常流转

```text
待接单
→ 已取消
```

```text
待接单
→ 已拒单
```

### 堂食状态说明

| 状态 | 说明 | 用户端展示 | 商家端可操作 |
|---|---|---|---|
| 待接单 | 用户已提交订单，等待商家处理 | 等待商家接单 | 接单、拒单 |
| 已接单 | 商家已接单 | 商家已接单 | 开始制作 |
| 制作中 | 商家正在制作 | 正在制作 | 标记待取餐 |
| 待取餐 | 商品已做好，等待用户取餐或用餐 | 待取餐 | 完成订单 |
| 已完成 | 订单完成 | 已完成 | 查看详情 |
| 已取消 | 用户取消订单 | 已取消 | 查看详情 |
| 已拒单 | 商家拒绝订单 | 已拒单 | 查看详情 |

## 16. 原型设计要求

### 16.1 视觉风格

- 主色：橙色
- 辅助色：白色、浅灰色
- 风格：简洁、温暖、生活化
- 卡片：圆角、轻阴影
- 按钮：主按钮使用橙色
- 危险操作：红色
- 成功状态：绿色
- 页面布局：符合微信小程序习惯

### 16.2 尺寸建议

- 画板尺寸：375px × 812px
- 页面左右边距：16px
- 卡片圆角：12px
- 主按钮高度：44px到48px
- 输入框高度：44px
- 卡片内边距：16px
- 常用间距：8px、12px、16px、24px

## 20. 后续开发建议

第一版建议优先开发以下闭环：

```text
用户下单
→ 商家接单
→ 商家制作
→ 堂食待取餐或配送中
→ 完成订单
```

不要一开始加入太多扩展功能。

建议开发顺序：

1. 数据库表设计
2. 用户登录
3. 商家登录
4. 商品管理
5. 用户商品浏览
6. 购物车
7. 下单
8. 商家接单
9. 订单状态流转
10. 用户订单查询
11. 店铺营业状态
12. 异常状态处理

## 21. 总结

本项目第一版的核心不是做成大型外卖平台，而是做成一个小而完整的点餐系统。

核心闭环是：

```text
用户选择堂食或配送
→ 用户点餐下单
→ 商家接单
→ 商家制作
→ 堂食待取餐或商家自配送
→ 订单完成
```

只要这个流程稳定清楚，后续再扩展支付、提醒、评价、会员、配送员等功能会更加容易。
